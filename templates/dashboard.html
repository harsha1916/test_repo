‚ÑπÔ∏è Entry/Exit Tracking:
When enabled, the system tracks each card's last scan. A transaction is only created when:
  • The card is scanned for the first time (no transaction yet)
  • The time since last scan exceeds the minimum gap (transaction created)
This prevents duplicate transactions when someone stays in the area and triggers the reader multiple times.

Configure authentication methods for API access.

‚ÑπÔ∏è About Basic HTTP Authentication:
  • Enabled: API accepts both token-based and Basic Auth (username:password)
  • Disabled: Only token-based authentication is accepted
  • Basic Auth can be used in URLs: http://username:password@host/api
  • Or in headers: Authorization: Basic base64(username:password)
‚ÑπÔ∏è About Wiegand Formats:
26-bit: Standard format used by most proximity cards (125kHz). Supports 8-bit facility code and 16-bit card number.
34-bit: Extended format for larger installations. Supports 16-bit facility code and 16-bit card number.

Check your card manufacturer's specifications or test both formats to determine which one your cards use.
